## Reflection on Software Development Principles

### Interpreting User Needs and User Stories
Understanding user needs is one of the most important parts of software development. I interpret user needs by analyzing requirements, observing how users interact with systems, and breaking complex requests into smaller functional components. Communication with stakeholders is essential because requirements are not always fully defined at the beginning of a project.

Creating user stories helps translate technical work into user-centered goals. User stories provide a simple way to describe what the user wants to accomplish, why it matters, and how success will be measured. They also help developers focus on delivering value rather than just completing technical tasks. User stories support iterative development because they allow features to be planned, implemented, and tested in smaller increments.

### Approaching Program Development and Agile Processes
When developing programs, I prefer an organized and iterative approach. I start by understanding requirements, designing a basic architecture, and then implementing features step by step. Testing is done continuously rather than waiting until the end of development.

Agile processes are valuable because they promote flexibility and continuous improvement. In future development work, I hope to incorporate practices such as sprint planning, regular code reviews, and incremental releases. Agile methods allow developers to respond to changing requirements while maintaining product quality. Short development cycles and frequent feedback help ensure the software remains aligned with user expectations.

### Being a Good Team Member in Software Development
Being a good team member means communicating clearly, respecting different perspectives, and contributing positively to shared goals. Collaboration is essential in software development because most modern projects are too complex for one person to complete alone.

A strong team member writes clean and maintainable code, participates in discussions, helps review others’ work, and is open to feedback. Reliability and accountability are also important qualities. Supporting teammates, sharing knowledge, and working toward project success are key parts of professional software development.
